# Superpowers vs Gstack 全方位对比

## 核心定位

| | Superpowers | Gstack |
|---|---|---|
| **创始人** | obra（独立开发者） | Garry Tan（YC CEO） |
| **定位** | "全能首席架构师+项目经理" | "一支完整的虚拟工程团队" |
| **设计哲学** | **流程驱动** — 先干什么、后干什么 | **角色驱动** — 谁来干什么 |
| **Skill数量** | 20+ | 28 |

## 工作流哲学

**Superpowers**: 需求 → 设计文档 → Git分支 → 计划拆解 → TDD循环 → 子代理执行 → 双重审查 → 合并
**Gstack**: office-hours(质疑) → CEO复盘(战略) → Eng复盘(架构) → 设计Review → 实现 → QA(browse真测) → ship → retro

## 各擅长场景

- **选 Superpowers**：正经软件工程、TDD、代码审查、自动化流程、独立开发者
- **选 Gstack**：产品/创业视角、真实浏览器测试、YC模式、角色扮演交互

## 能否同时装

理论上不冲突，但可能打架（流程重复、行为不一致）。建议分管不同阶段（如 Gstack 管产品设计，Superpowers 管工程实现）。

## 一句话总结

> Superpowers = **工程纪律驱动的 AI 编程方法论**（流程优先）
> Gstack = **角色扮演驱动的 AI 工程团队**（人设优先）

## 关键 Skill

- Superpowers: `subagent-driven-development`, `systematic-debugging`, `test-driven-development`
- Gstack: 各角色 slash commands（/office-hours, /plan-ceo-review, /retro 等）
